# 📦 AWS IAM User Management & Authentication using Terraform

## 📖 Overview

This project demonstrates how to build a scalable, secure, and production-grade IAM automation system using Terraform.

It automates:
- IAM user creation
- Role-Based Access Control (RBAC)
- Secure login profile setup
- MFA enforcement
- Audit logging

The system is designed to scale from 10 users to 1000+ users using a CSV-driven approach.

## 🎯 Key Objectives
- Implement Infrastructure as Code (IaC) using Terraform
- Automate IAM user provisioning from CSV
- Enforce least privilege access using RBAC
- Ensure secure authentication (MFA + encrypted passwords)
- Enable audit logging with CloudTrail
- Follow industry best practices

## 🧠 Architecture Flow
CSV → Terraform → IAM Users → Groups → Policies → Attachments → MFA → CloudTrail

## 🛠️ Technologies Used
```plaintext
Terraform (>= 1.5)
AWS Provider (~> 5.x)
AWS IAM
AWS S3 (Remote Backend)
AWS CloudTrail
GPG (for password encryption)
```

# 🧩 Features

## 👤 **IAM** User Automation
- Dynamic user creation using `for_each`
- Unique identifier: `employee_id`
- Username: `firstname.lastname`
- `Username can be upgrade to firstname.lastname.employee_id`

## 👥 **RBAC** (Role-Based Access Control)
| Role      | Group             |
| --------- | ----------------- |
| Developer | dev-group         |
| Tester    | testing-group     |
| DevOps    | operations-group  |
| Manager   | management-group  |

## 🔐 Security Features
- Passwords generated by **AWS** (via login profile)
- Encrypted using **PGP**
- Mandatory password reset on first login
- **MFA** enforcement for all users

## 📜 Audit Logging
- Enabled using **AWS** CloudTrail
- Logs:
  - Login events
  - **API** calls
  - Resource changes

## 📦 Remote State Management
- S3 backend with locking enabled (`use_lockfile = true`)

# 🚀 Setup Instructions

## 🔹 1. Prerequisites
- Terraform installed
- **AWS CLI** configured
- **GPG** installed

## 🔹 2. Configure **AWS**
aws configure 
- Ensure region matches:
```plaintext
ap-south-1 default region set to ap-south-1.
```

## 🔹 3. Create S3 Backend Bucket
aws s3api create-bucket --bucket prudhvi-tf-state-unique-12345 --region ap-south-1 --create-bucket-configuration LocationConstraint=ap-south-1 
enable versioning:
aws s3api put-bucket-versioning --bucket prudhvi-tf-state-unique-12345 --versioning-configuration Status=Enabled

## 🔹 4. Generate **PGP** Key
gpg --full-generate-key  RSA and RSA, 2048 or 4096 bits 
f.e., export:
gpg --export --armor your_email@example.com > public_key.asc  (Note: Replace `your_email@example.com` with your actual email.)

## 🔹 5. Initialize Terraform
- terraform init
- terraform fmt
- terraform validate 

## 🔹 6. Deploy Infrastructure
- terraform plan
- terraform apply --auto-approve

# 🔐 Accessing **IAM** Users
decode password:
echo *<encrypted_password>* | base64 --decode | gpg --decrypt 
- sign in to **AWS** console, 
- change password, 
- enable **MFA**.

# 🧪 Testing
- test by logging in as different users, 
- verify permissions per role, 
- check **MFA** enforcement, 
- validate logs in CloudTrail.

# 🧹 Cleanup
Use `terraform destroy` or `terraform destroy --auto-approve` to remove all resources.

# 📊 Outputs
display **IAM** usernames, group mappings, policy ARNs, encrypted passwords.

# 🧠 Key Learnings

Topics covered in this project include:
- Terraform `for_each` with maps for scalable design
- **IAM RBAC implementation**
- Handling **AWS provider v5 limitations**
- Secure password management with **PGP**
- Importance of remote state and state locking

> **Important Notes:**
> - Do **NOT** commit private keys.
> - **IAM** is global (region does not affect IAM).
> - MFA is enforced via policy (not a native IAM setting).
